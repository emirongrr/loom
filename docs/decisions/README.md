# Engineering Decisions

Decision records are required only for changes that cross the threshold in
`docs/project/engineering.md`. Keep each record short, evidence-based, and
committed with the implementation.

Name records `NNNN-short-title.md` and use this template:

```md
# Decision title

Status: proposed | accepted | superseded
Date: YYYY-MM-DD

## Problem

What user, security, or operational problem must be solved?

## Evidence

What measurements, tests, incidents, or constraints inform the decision?

## Options

Which realistic options were considered and why were they rejected or kept?

## Decision

What is changing, and what measurable acceptance condition must pass?

## Residual risks

What remains unproven, trusted, unavailable, or reversible?
```
