# SDK

Loom currently ships two early SDK packages: `@loom/account` for account
lifecycle intent builders and `@loom/privacy` for a Kohaku-compatible privacy
runtime boundary.

These packages are wallet-engine primitives, not a finished wallet client.
They must not hide trust assumptions, select default infrastructure, or claim
production privacy before protocol-specific review and live rehearsal.

## Account lifecycle

`@loom/account` builds explicit intents for account deployment, session grant
and revoke, recovery proposal, migration, vault withdrawal, private vault
withdrawal binding, and optional paymaster policy.

The package deliberately avoids side effects:

- no default RPC;
- no default bundler;
- no default paymaster;
- no transaction sending;
- no Loom-operated recovery coordinator.

Wallets remain responsible for explaining the intent, collecting signatures,
simulating where appropriate, and publishing through user-selected
infrastructure.

## Privacy runtime

The runtime gives wallet builders a place to host Kohaku plugins behind:

- explicit provider consent;
- metadata-budget checks;
- local storage boundaries;
- no default RPC query before user consent;
- local private scan lifecycle state;
- a shielded-pool adapter wrapper;
- a Railgun profile boundary;
- future privacy-pools, stealth, and Aztec adapter surfaces.

## Provider profile

```ts
import {
  createConsentStore,
  createKohakuHost,
  createProviderProfile,
} from '@loom/privacy'

const profile = createProviderProfile({
  mode: 'user-rpc',
  chainId: 1,
  endpoint: 'https://rpc.example',
  verified: false,
  metadataBudget: {
    protocol: 'railgun',
    chainId: 1,
    items: [
      {
        surface: 'rpc',
        reveals: 'target chain and request timing',
        required: true,
        mitigation: 'user-selected endpoint',
      },
    ],
  },
})
```

## Consent before network access

```ts
const consent = createConsentStore()
const host = createKohakuHost({
  providerProfile: profile,
  consentStore: consent,
  metadataPolicy: {
    allowedSurfaces: ['rpc'],
    requireKnownMitigation: true,
  },
})

// Throws until the wallet grants consent.
await host.network.fetch(profile.endpoint)
```

## SDK package roadmap

| Package | Purpose | Status |
| --- | --- | --- |
| `@loom/account` | Account deploy, session grant/revoke, recovery proposal, migration, vault withdrawal, private vault binding, and paymaster policy builders. | Implemented boundary |
| `@loom/privacy` | Kohaku host, provider profiles, metadata budgets, local scan lifecycle, shielded-pool wrapper, and Railgun profile boundary. | Implemented boundary |
| `@loom/client` | Wallet engine for mobile apps and partner wallets with verified state and clear signing. | Planned |

SDKs should improve UX without becoming account authority.
