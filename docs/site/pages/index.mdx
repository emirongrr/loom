Loom is not a closed wallet application. It is an open wallet engine: immutable
accounts, passkeys, social recovery, bounded permissions, vault safety, L1-rooted
identity direction, and a Kohaku-backed privacy SDK boundary.

## Principles

- **No admin keys:** no proxy upgrade path, factory authority, or developer recovery path.
- **Passkeys first:** hardware-backed credentials instead of seed phrases for daily use.
- **Recovery without custody:** guardians can recover or cancel but cannot spend.
- **Privacy as security:** only the user's local wallet should need the global account graph.
- **Permissionless exit:** users can migrate clients, validators, guardians, infrastructure, and future account systems.
- **Verification over trust:** RPCs, indexers, relayers, and paymasters are convenience layers, not authority roots.

## The walkaway test

If Loom's frontend, RPC, bundler, paymaster, indexer, notification service, and
recovery coordinator disappear, users should still be able to discover account
state, verify it, recover access, publish authorized operations, and migrate away
with independent software.

That test is the product.
