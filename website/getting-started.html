Build from source

Start with the contracts, then add client layers.

Loom is currently a pre-audit contract and SDK-boundary repository. The first integration step is to understand account authority, validators, recovery, sessions, and the privacy host boundary.

Install and verify

git clone https://github.com/emirongrr/loom.git
cd loom
npm ci
npm run verify:quick

Use Node.js 22 and Foundry v1.7.1. The quick verification path includes docs, website checks, privacy SDK tests, formatting, linting, size checks, gas snapshots, contract tests, and source-policy checks.

Run privacy SDK boundary tests

npm run privacy:test

This validates that the Kohaku host runtime does not query any provider before explicit user consent and metadata-budget approval.

How to launch a wallet on Loom

Step Engine responsibility Client responsibility
1. Account creation Deploy immutable LoomAccount with selected validators and guardian root. Create passkeys, explain recovery, verify deployment bytecode, and store local metadata.
2. Daily access Validate passkey or MFA operations under policy and hook constraints. Build clear calls, simulate risk, route through user-selected infrastructure.
3. Permissions Enforce bounded session authority and immediate revocation. Translate app requests into exact permissions the user can understand.
4. Recovery Require guardian threshold, visibility, delay, expiry, and cancellation. Coordinate guardian ceremony without becoming a recovery custodian.
5. Privacy Keep core usable without privacy providers. Use Kohaku-backed adapters, local scanning, and metadata budgets.
