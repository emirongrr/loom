Pre-audit security posture

Security claims must be evidence-backed.

Loom is security-critical wallet infrastructure. The project is pre-audit and should not secure production funds. The goal of this page is to make assumptions and missing evidence visible.

Protected

Developer rug paths, factory authority, unsupported execution modes, duplicate guardians, stale config replays, and unbounded sessions.

Still required

Independent audit, live multi-bundler tests, browser WebAuthn fixtures, testnet rehearsals, production proof verifiers, and bug bounty.

Never allowed

Mandatory Loom RPC, mandatory paymaster, admin upgrade key, global public identity registry, or privacy provider as account authority.

Production gates

Gate Status Why it matters
Independent audit Missing Immutable accounts cannot be patched after deployment.
Browser/device WebAuthn fixtures Missing Passkey compatibility must be proven across real authenticators.
Live bundler qualification Missing ERC-4337 liveness must not depend on one hosted path.
Privacy adapter review Pending Zero-knowledge protocols still leak metadata through providers, relayers, timing, and scanning choices.
L1 proof verifier audit Pending Cross-chain authority must be proven, not trusted through a service.
