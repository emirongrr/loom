Contracts-first wallet engine for Ethereum

Build wallets users can leave without losing power.

Loom is an open account stack for passkey wallets, social recovery, bounded permissions, vault safety, L1-rooted identity, and Kohaku-backed privacy clients. It is not a closed wallet app. It is infrastructure for teams that want fintech-grade UX without surrendering user sovereignty.

No admin keys

No proxy upgrade path, no factory control, no mandatory module registry, and no Loom-operated service with account authority.

Passkeys and recovery

P-256/WebAuthn, multi-passkey threshold authentication, guardian recovery, emergency freeze, and delayed cancellation paths.

Privacy as security

Kohaku is the SDK privacy runtime boundary. Provider access requires consent and metadata-budget checks before network use.

What Loom provides today

The current repository is pre-audit contract infrastructure plus the first privacy SDK host boundary. Production wallets still need independent audits, live bundler evidence, browser passkey fixtures, and concrete privacy adapters.

Walkaway test

If Loom's frontend, RPC, bundler, paymaster, indexer, and recovery coordinator disappear, users should still be able to discover state, verify it, recover access, publish authorized operations, and migrate away with independent software.

Pre-audit

Loom is not production-ready. Do not secure production assets until independent audits, live deployment rehearsals, browser WebAuthn fixtures, and release gates are complete.

Loom is MIT licensed wallet infrastructure. Build useful interfaces, but keep exit real.
